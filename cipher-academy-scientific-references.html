Scientific References

This document summarizes the research that informs Cipher Academy's design. The goal is not to replicate academic protocols exactly, but to draw from established findings in cognitive science and apply them to a simple, offline learning experience.

Cipher Academy is research-informed, not research-prescriptive. The systems described below are inspired by scientific work, but adapted for usability, short sessions, and self-paced learning.

Part 1 — Brain & Learning Benefits

Learning encoding systems such as Morse code or Braille engages multiple cognitive systems, including language processing, working memory, and perceptual recognition.

Neuroimaging studies show that even short training periods can produce measurable changes in the adult brain:

  • Schlaffke et al. (2017): found structural changes in white matter after Morse learning, correlated with decoding performance.
  • Junker et al. (2023): showed that Morse decoding activates working memory, long-term memory retrieval, and language-related brain regions.
  • Siuda-Krzywicka et al. (2016): demonstrated large-scale cortical reorganization in sighted adults learning Braille.

These findings suggest that decoding symbolic systems is not just a puzzle activity, but a form of cognitive training involving pattern recognition, memory, and language-like processing.

Design interpretation

Cipher Academy draws inspiration from this research by combining multiple forms of input:

  • Visual decoding: symbols such as Pigpen or Braille
  • Auditory decoding: Morse code
  • Interactive/tactile-like input: Braille Touch mode

This multimodal approach is intended to reinforce learning across different sensory channels.

However, the app is not a controlled training protocol, and these design choices are adaptations rather than direct implementations of experimental methods.

Part 2 — Adaptive Letter Selection

During a session, Cipher Academy selects which letter to show next using a dynamic weighting formula based on:

  • current mastery within the session
  • long-term retention (spaced repetition)
  • recency (spacing)
  • past errors (confusion tracking)
  • similarity between symbols (pairing)

This approach is inspired by several lines of research:

  • Mettler, Massey & Kellman (2016): adaptive sequencing in perceptual learning
  • Kornell & Bjork (2008): interleaving vs. blocking
  • Carvalho & Goldstone (2015): discriminative contrast through sequencing
  • Settles & Meeder (2016): modeling long-term memory (half-life regression)
  • Metcalfe (2017): learning from errors
  • Pyc & Rawson (2009): strengthening through successful retrieval

Formula overview

Each letter is assigned a weight:

Wᵢ = Eᵢ · Mᵢ · Bᵢ · Sᵢ · Rᵢ · (1 + α Ĉᵢ) · (1 + β K̂ᵢ Rᵢ)

Where:

  • Bᵢ: long-term mastery (SRS box)
  • Sᵢ: session progress (how close the letter is to being mastered)
  • Rᵢ: recency (time since last seen)
  • Cᵢ: accumulated confusion involving this letter
  • Kᵢ: confusion with the previously shown letter (contrast effect)
  • Mᵢ: reduces weight of already-mastered letters
  • Eᵢ: prevents immediate repetition of the same letter (UX rule)

Design interpretation

This formula is not taken from a single paper. Instead, it combines several well-supported principles:

  • Spacing: improves retention
  • Interleaving: improves discrimination between similar items
  • Errors: reveal what needs reinforcement
  • Contrast between similar items: accelerates perceptual learning

The result is a simple, deterministic system that adapts in real time without requiring external data or machine learning.

Part 3 — Level Design & Progression

Cipher Academy's progression system is partially inspired by established learning methods, particularly for Morse code.

  • Koch (1936): introduced a method where learners start with a small set of characters and only add new ones after reaching high accuracy.
  • The app follows this principle by introducing letters in small batches and requiring mastery before progressing.

However, the implementation is adapted:

  • Letter order is chosen for **perceptual contrast**, not historical sequence
  • Multiple ciphers are used to engage different cognitive processes
  • Previously learned letters remain in the pool, creating **interleaved practice**

Design interpretation

The progression system is inspired by research, but not a strict reproduction of any single method. It is designed to balance:

  • learnability (small steps)
  • retention (repetition over time)
  • usability (short, self-contained sessions)
